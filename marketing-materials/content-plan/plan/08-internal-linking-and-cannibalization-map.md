# Internal linking and cannibalization map

## Architecture

`/deals/arraysubs/` → resources hub → category hub → editorial pillar → supporting articles → relevant use case/feature/comparison → pricing. Every supporting article also links to the exact existing recipe/manual page when the reader is ready to configure ArraySubs.

## Global rules

- Use descriptive anchors; avoid repeated exact-match anchors sitewide.
- Add links inside the relevant explanation, not in a generic “related posts” footer only.
- One article should normally contain 4-8 useful internal links, not an arbitrary quota.
- Product and comparison pages should link back to the most helpful neutral guides, formulas, and migration articles.
- If two drafts answer the same dominant question with the same format, merge them before publishing.
- Run quarterly Search Console query overlap checks; consolidate pages that compete for the same intent.
